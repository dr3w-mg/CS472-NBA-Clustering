Archetypes

We used a Gaussian Mixture Model to soft-assign 322 NBA players into 9 distinct playing archetypes, then matched those archetypes to team needs to display the best potential players for each roster. This project was inspirered by the paper "Mapping the Modern NBA: Discovering Player Archetypes Through Data-Driven Clustering" and the codebase was refactored for the GMM approach based off this GitHub repository.

Pipeline

How It Works

From raw NBA API data to roster recommendations in four steps.

01

Data Collection

Pulled traditional, advanced, and hustle stats from three nba_api datasets for the 2024–25 regular season.

02

Preprocessing

Filtered to players with ≥ 30 games and ≥ 15 min/game. Joined all three tables into one 322-row dataset with 26 standardized features.

03

GMM Clustering

Used BIC/AIC model selection to choose k = 9. A Gaussian Mixture Model soft-assigns players to clusters, capturing role overlap.

04

Team Matching

Built cluster distribution profiles for all 30 teams, compared against league averages, and ranked the top 10 players for each team by a match score against team needs.

Methodology

Modeling Decisions

Key choices made during feature engineering, model selection, and cluster assignment.

F Feature Engineering

26 features were organized into thematic groups: scoring & shot profile, playmaking, rebounding, defense, and hustle. All features were standardized with the scikit-learn library to prevent high-magnitude stats like points from dominating smaller-scale efficiency metrics.

C Selecting # of Clusters

We tested cluster counts from 3 to 10 using both BIC and AIC criteria to balance model fit against complexity. Both metrics converged on k = 9, which produced distinct archetypes without over-fragmenting.

G Gaussian Mixture Model

Unlike k-means, a GMM produces soft cluster assignments (a probability distribution across all 9 archetypes for each player). This reflects reality: a player's role varies across lineups, opponents, and game situations. A player can be 60% "3-and-D Wing" and 30% "Secondary Scorer."

T Team Need Scoring

Each team's cluster distribution was compared to the league average to identify gaps. Players were then ranked by how well their soft cluster probabilities filled those specific team needs, highlighting players who would add complementary, not redundant, value.
